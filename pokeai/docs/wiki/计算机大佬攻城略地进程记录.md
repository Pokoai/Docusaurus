~~2. b 站黑马视频：前 100 集是关于 Linux 基本操作的，对于我来说，反而是被这部分吸引了。后面的 python 部分感觉太过小白，我换别的了。因为我已经有 python 基础了，但是不太系统，又有些时间没接触了。所以想找个精炼相对系统的视频帮我巩固下之前学习的效果。其实我最大的问题是，之前学习完基础部分算入门后，没有乘胜追击，去做一两个实战项目，导致后面很容易忘记，而且没有信心。又得重复来学习下基础。~~

**这也是一个教训，对于编程语言的学习，以及计算机底层基础知识的学习，都有一个至关重要的点（我后来发现的）就是：学习完后一定要做项目实战，才算真正学完入门。否则放一段时间不接触就全忘记了，相当于白学。**
   
3. b站 马士兵Python教学

可能你会觉得看视频太多浪费时间，没有看书来的快。我之前也是这个想法，而且我还能耐得住性子看完一整本五六百页的书，所以一直觉得我看书自学就 ok 了。但是近期我才发现，我对自己的认知错误了，一开始就看书自学一个新领域，虽然我可以看完书（包括做课后习题敲代码实践），前期掌握的是不错，也有信心。但是呢，过一段后没接触后就发现沉淀下来的特别少，记忆点也很少。

所以我才换了一种自学方式，入门阶段先看视频，听老师讲课，刺激性更大，对于我来说记忆点更深刻。看完视频后再看经典书籍巩固一遍，知识留存率更大一些。这是针对非科班、计算机底子相对较薄的自学者来说的。视频中也许老师无关紧要的某句提醒，反而对我们作用更大，因为跟知识点无关的话书上是不会写的。

另外呢，最好加上总结输出，我的方式是：学习完做几个小项目练练手，以此为基础做分享输出。学习过程中也可以输出。这几个方式配合起来，知识留存率已经远超其他人了，对于我们喜欢搞懂底层原理的人来说，可以达到目的了。

对于基础很扎实的计算机学习者，可以直接看书自学新领域。这也是我后期的方式。
